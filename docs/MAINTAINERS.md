## Becoming a Maintainer

We welcome contributions! To become a maintainer:

1. Make significant contributions to the project
2. Demonstrate understanding of the codebase
3. Show commitment to project goals
4. Be nominated by existing maintainers
5. Be approved by the core team

## Responsibilities

Maintainers are expected to:
- Review and merge pull requests
- Triage issues
- Participate in release planning
- Maintain code quality standards
- Help guide project direction
